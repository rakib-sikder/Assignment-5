What is the Document Object Model (DOM)?

The DOM stands for the Document Object Model. It’s like the backstage pass for web pages. When a web page loads in your browser, the DOM is quietly constructed behind the scenes. Imagine it as a tree of interconnected objects that represent the structure and content of the page.each HTML element (like headings, paragraphs, images, buttons) becomes an object in this tree. The root of the tree is the entire HTML document, and from there, branches extend to individual elements. JavaScript can then interact with these objects, making your web page dynamic. It can change the content of HTML elements. Want to update that button text? Done! It can tweak HTML attributes. Change an image source or adjust a link’s target. It can play with CSS styles. Turn that bland paragraph into a stylish masterpiece. It can add or remove HTML elements. Say hello to dynamically generated content! It can even react to HTML events. Clicks, hovers, and more—JavaScript’s got it covered.

How do you select an Element from the DOM?

If an element has a unique id attribute (and it should be unique, like a snowflake), you can grab it using document.getElementById('yourElementId'). It’s like calling out someone by their full name at a party—no confusion! Example:const myElement = document.getElementById('myUniqueButton'); When you’ve got a bunch of elements with the same class (like a herd of sheep), you can round 'em up using document.getElementsByClassName('yourClassName'). Note: This returns a collection (like a sheep pen), so you’ll need to loop through it or pick a specific index. Example:const buttons = document.getElementsByClassName('myButtons');

what is event delegation in the contex of the DOM, why is useful?

Event delegation is a pattern that leverages the concept of event bubbling. When an event (like a click or keypress) occurs on an element, it doesn’t just affect that specific element; it also “bubbles up” through its parent elements in the DOM hierarchy. Instead of attaching event listeners directly to individual child elements, event delegation allows you to handle events at a higher level—typically on a parent or ancestor element. This parent element then becomes responsible for handling events triggered by its descendants By default, when the button receives an event (e.g., a click), that event bubbles up through the DOM tree.
